project-starter

A reusable template for AI-assisted development with built-in documentation freshness

Create a New Project

gh repo create MyProject --template Brenton-Thriviti/project-starter --clone
cd MyProject

What You Get Immediately

  • 6 AI coding tools pre-configured (Claude, Cursor, Copilot, Gemini, Codex, Windsurf)
  • Spec-driven development workflow
  • Documentation freshness system with expiry dates
  • Interactive git helper for teams
  • GitHub issue/PR templates
  • Editor configuration (.editorconfig, VS Code settings)

First Steps After Cloning

  1. Run /speckit.constitution to review project principles
  2. Update .speckit/memory/team.md with team members
  3. Start building with /speckit.specify

What is Spec-Kit?

Spec-Kit is GitHub's toolkit for spec-driven development. Instead of jumping straight to code, you define what you're building first, then let AI help implement it.

Available Commands

Command Purpose
/speckit.constitution Define project principles and guidelines
/speckit.specify Create a specification for what you want to build
/speckit.plan Generate an implementation plan from your spec
/speckit.tasks Break the plan into actionable tasks
/speckit.implement Execute tasks to build your feature
/speckit.checklist Generate a verification checklist

Workflow

constitution → specify → plan → tasks → implement

RFC Pattern + Expiry Headers

Every document in .speckit/ uses YAML frontmatter to track freshness:

---
id: SPEC-001-feature-name
type: spec
created: 2025-01-02
last_updated: 2025-01-02
review_by: 2025-04-02
status: active
---

Document Types

Type Mutable? Review Period
spec Yes 90 days
gap-analysis No (immutable) 30 days
research No (immutable) 180 days
memory Yes (sparingly) Annual

Interactive Git Helper

No git commands to remember. Just run /git and choose what you want to do:

Branch: main
Changes: 3 files modified

What would you like to do?
  1. Create a new branch
  2. Save my work (commit)
  3. Push to remote
  4. Get latest from main

Branch Naming

Branches follow the pattern: {username}/{description}

  • brenton/add-login-feature
  • ryan/fix-modal-bug
  • scott/update-readme

Team Configuration

Edit .speckit/memory/team.md to add team members for the branch selector.

Issue Templates

Located in .github/ISSUE_TEMPLATE/:

  • bug_report.md - Structured bug reports with reproduction steps
  • feature_request.md - Feature proposals with use cases
  • config.yml - Template chooser configuration

Pull Request Template

.github/PULL_REQUEST_TEMPLATE.md includes a checklist for code review.

CODEOWNERS

.github/CODEOWNERS auto-assigns reviewers based on file paths.

Tracking Customizations

Upgrading Spec-Kit

When new spec-kit features are released:

./scripts/upgrade-speckit.sh

This script:

  1. Fetches latest spec-kit templates
  2. Shows diff of changes
  3. Lets you selectively merge updates

When to Upgrade

  • New AI tools are added to spec-kit
  • Important bug fixes or improvements
  • New commands you want to use
